# 2255_devoir


1, enregistrer mon bac.  
Acteur: résident (principal)
Préconditions: résident doir avoir un code QR sur son bac
Postcondition:	la système doit enregistrer les info du bac
Scénario principal:
	1, résident sélectionne un type de bac qu'il veut enregistrer
	2, donne un nom à son bac
	3, scanne la code QR du son bac
	4, système vérifie les infos 
	5, système affiche la page de réussi
	

2, afficher les métriques 
Acteur: résident (principal)
Préconditions:	résident doit avoir un compte
Postconditions:	aucun
Scénario principal:
	1, résident sign in
	2, résident consulte les métriques
	3, Système affiche les métriques
	

	

3, voir les activités que je participe
Acteur: résident (principal)
Préconditions:  résident doit avoir un compte
Postconditions: aucun
Scénario principal:
	1, résident sign in
	2, résident consulte les activités qu'il participe
	3, Système affiche les activités qu'il participe




4, Trouver un consommateur 
Acteur: résident (principal)
Préconditions: résident doit avoir un compte
Postconditions: aucun
Scénario principal:
	1, résident consulte les consommateurs
	2, système affiche une page de consommateurs avec un mécanisme de recherche
	3, résident clique le consommateur qu'il veut contacter
	4, système affiche les infos plus précis sur ce consommateur
	


5, crée un compte 
Acteur:	résident ou consommateur (principal)
Préconditions: aucun
Postconditions: système doit enregistrer les infos sur ce nouveau compte
Scénario principal:
	1, résident crée son propre Username et mot de pass
	2, Système affiche une formulaire
	3, user remplit la formualaire
	4, user remet la formulaire
	5, système affiche la page de réussi

Scénario alternative:
	5.a, quelques infos sont manquées ou invalides
		5.a.1, user remplit à nouveau les infos
		5.a.2,  user remet la formulaire

9, Mise à jours les 'usage du bac
Acteur: sensor(principal) municipal(secondaire)
Pré:	les sensors sont installées dans les bacs
Post:	la page de l'état du bac doit être mise à jours
Scénario principal:
	1, sensor scanne le bac
	2, sensor envoie les infos collectés au municipal
	3, municipal analyse les infos
	4, municipal élabore les métriques
	5, municipal mise à jours la page de l'état du bac orienté aux résidents
	6, système affiche la page de réussi



10, Mise à jours le récupération et livraison des déchets
Acteur: camion (principal), municipal(secondaire)
Pré: camion doit arriver un checkpoint de récupération ou livraison
post: les métriques doivent être mise à jours
Scénario principal:
	1, camion envoie les infos sur les déchets dès qu'il arrive un checkpoint
	2, municipal analyse les infos
	3, municipal élabore les métriques
	4, municipal mise à jours les métriques
	5, système affiche la page de réussi




11, Mise à jours les métriques sur la traitement de déchets
Acteur: consommateur (principal), municipal(secondaire)
Pré:  aucun
post: les métriques sur les activités, traitement de déchets doivent être mise à jours
Scénario principal:
	1, consommateur demande une mise à jours sur les métriques dès qu'il commence à utiliser les déchets
	2，système affiche une formulaire
	3, consommateur remplit la formulaire
	4, consommateur remet la formulaire
	5, municipale vérifie la formulaire
	6, municipale mise à jours les métriques 
	7, municipale mise à jours la page d'activité

Scénario alternatif:
	5.a, municipale n'approuve pas la formulaire
		5.a.1, consommateur remplit à nouveau la formulaire
		5.a.2, consommateur remet la formulaire
